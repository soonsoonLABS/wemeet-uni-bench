# 평가 기준 설계

> 대학생 관점에서 AI 모델의 실질적 유용성을 측정하기 위한 평가 체계 초안

---

## 평가 카테고리

### 1. 📝 학술 글쓰기 (Academic Writing)
- 레포트 구조화 및 작성
- 논문 요약 및 비평
- 참고문헌 기반 서술
- 한국어 학술 문체 적합성

### 2. 🔢 이공계 문제 풀이 (STEM Problem Solving)
- 수학 (미적분, 선형대수, 확률통계)
- 물리/화학/재료공학 계산 문제
- 단계별 풀이 과정 제시 능력
- 수식 표현 정확도

### 3. 💻 코딩 과제 (Programming Tasks)
- 자료구조/알고리즘 구현
- 과제 수준 프로그래밍 (Python, C, Java)
- 코드 설명 및 디버깅
- 테스트 케이스 생성

### 4. 🔍 리서치 보조 (Research Assistance)
- 연구 질문 구체화
- 관련 논문/자료 탐색 제안
- 실험 설계 조언
- 데이터 분석 방법론 제안

### 5. 📊 데이터 분석 (Data Analysis)
- 데이터 해석 및 인사이트 도출
- 시각화 방법 제안
- 통계 분석 수행
- 결과 서술

### 6. 🗣️ 학습 보조 (Learning Support)
- 개념 설명 (다양한 수준)
- 오개념 교정
- 예시 생성
- 시험 대비 질의응답

## 난이도 체계

| 레벨 | 설명 | 대상 |
|------|------|------|
| L1 | 교양/기초 과목 수준 | 1학년 |
| L2 | 전공 기초 수준 | 2학년 |
| L3 | 전공 심화 수준 | 3학년 |
| L4 | 졸업 프로젝트/대학원 입문 | 4학년 |

---

## 채점 기준 (Rubric)

### 정확성 (Correctness) — 40%
- 5: 완벽히 정확한 답변
- 4: 사소한 오류 1개 이하
- 3: 핵심은 맞으나 부분 오류
- 2: 방향은 맞으나 주요 오류
- 1: 대부분 부정확

### 유용성 (Helpfulness) — 30%
- 5: 즉시 과제에 활용 가능
- 4: 약간의 수정으로 활용 가능
- 3: 참고 수준
- 2: 방향만 제시
- 1: 도움 안 됨

### 설명력 (Explainability) — 20%
- 5: 단계별 명확한 설명, 학습에 도움
- 4: 대부분 이해 가능한 설명
- 3: 설명 있으나 불충분
- 2: 결과만 제시
- 1: 이해 불가

### 한국어 품질 (Korean Quality) — 10%
- 5: 자연스러운 학술 한국어
- 4: 약간의 어색함
- 3: 이해 가능하나 번역투
- 2: 상당히 부자연스러움
- 1: 한국어 부적합

## 문제 형식

각 문제는 다음 구조를 따릅니다:

```yaml
id: "AW-L2-001"
category: "academic_writing"
level: 2
title: "연구 보고서 서론 작성"
prompt: |
  다음 주제에 대한 연구 보고서의 서론을 작성하세요...
context: |
  (필요시 제공되는 배경 자료)
expected_output: |
  (모범 답안 또는 채점 기준)
scoring:
  correctness: "..."
  helpfulness: "..."
  explainability: "..."
  korean_quality: "..."
```
