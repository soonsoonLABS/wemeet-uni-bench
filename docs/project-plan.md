# 프로젝트 계획서

## 1. 프로젝트 배경

### 문제 인식
- 기존 LLM 벤치마크(MMLU, HumanEval, GPQA, MT-Bench 등)는 AI 연구자 관점에서 설계됨
- 한국 대학생이 실제 학업에서 AI를 활용할 때 필요한 능력과 기존 벤치마크 측정 항목 간 괴리 존재
- 예: 한국어 학술 논문 요약, 공학 과제 풀이, 실험 데이터 분석, 레포트 작성 등

### 목표
대학생 관점에서 AI 모델의 실질적 유용성을 측정할 수 있는 벤치마크 체계를 설계하고, SAM 플랫폼의 모델들에 대해 실행하여 결과를 공개한다.

---

## 2. 연구 범위

### 조사 대상 기존 벤치마크
- **지식 평가**: MMLU, MMLU-Pro, GPQA
- **코딩**: HumanEval, MBPP, SWE-bench
- **수학/추론**: GSM8K, MATH, ARC
- **한국어**: KMMLU, KoBEST, CLIcK
- **대화/지시**: MT-Bench, AlpacaEval, Chatbot Arena
- **멀티모달**: MMMU, MathVista

### 도출할 평가 차원 (예시)
- 한국어 학술 텍스트 이해 및 요약
- 공학/이공계 문제 풀이 (수식 포함)
- 코드 작성 및 디버깅 (과제 수준)
- 리서치 질문 생성 및 문헌 분석
- 실험 데이터 해석 및 시각화 제안
- 레포트/에세이 구조화 및 작성 보조

---

## 3. 방법론

### Phase 1: 벤치마크 서베이
- 주요 벤치마크 논문 및 리더보드 조사
- 각 벤치마크의 평가 항목, 난이도, 한계점 정리
- 대학생 활용 시나리오와의 매핑

### Phase 2: 파라미터 설계
- 대학생 인터뷰/설문을 통한 니즈 파악
- 평가 차원 및 난이도 체계 설계
- 채점 기준(rubric) 수립

### Phase 3: 문제 제작
- 카테고리별 문제 제작 (최소 50문항 목표)
- 전문가(멘토) 검수
- 파일럿 테스트

### Phase 4: 벤치마크 실행
- SAM API를 통한 자동화된 벤치마크 실행
- 모델별 결과 수집 및 분석
- 시각화 및 리더보드 구성

### Phase 5: 결과 공개
- 최종 보고서 작성
- GitHub 공개 및 발표

---

## 4. 산출물

1. **벤치마크 서베이 보고서** — 기존 벤치마크 분석 문서
2. **평가 체계 설계서** — 파라미터, 카테고리, 채점 기준
3. **벤치마크 문제 세트** — 카테고리별 문제 및 정답/채점 기준
4. **실행 스크립트** — SAM API 연동 자동 실행 코드
5. **결과 보고서** — 모델별 성능 비교 분석
6. **최종 발표 자료**
