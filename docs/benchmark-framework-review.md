# 벤치마크 프레임워크 비판적 리뷰

> 작성일: 2025-05-23
> 대상: benchmark-framework-proposal.md (v0.1)
> 리뷰어: AI 비판적 분석 (SAM 키 만료로 단일 모델 수행, 추후 멀티모델 검증 예정)

---

## 1. 구조적 완결성

### 강점
X축 9개 도메인이 대학생 학업의 실제 상황을 잘 포괄하며, 특히 D9(언어/다국어)의 포함은 한국 대학생 맥락에서 차별적 가치가 있다.

### 약점/문제점

**Y축에서 빠진 것:**
- **시간 효율성(Efficiency)** — 같은 품질의 답을 얼마나 빨리 주는가. 학생은 마감이 있다. 30초 vs 3분은 체감 도움도에 큰 차이.
- **일관성(Consistency)** — 같은 문제를 10번 물었을 때 답의 분산. τ-Bench의 pass^k 지표가 이걸 측정했음. 한 번 잘 풀면 끝이 아니라 "항상 잘 푸는가"가 중요.
- **상호작용 능력(Interactivity)** — 학생이 "아 그게 아니라..."하고 수정했을 때 유연하게 대응하는가. 현재 C6(학습 유도)에 일부 포함되지만 별도 축으로 분리할 가치 있음.

**X축에서 빠진 것:**
- **취업/진로** — 대학생의 현실. 자소서, 면접 준비, 포트폴리오 피드백. D8(글쓰기)에 일부 포함되지만 성격이 다름.
- **시험 준비** — 중간/기말고사, 자격증, 수능(재수생). "문제 풀이 연습"은 D1~D5와 겹치지만 "시험 전략"은 별도 도메인.

**구조적 문제:**
- X축과 Y축이 완전히 독립적이지 않음. 예: D9(언어)와 C7(문체 적합성)은 상당 부분 겹침. D8(글쓰기)과 C5(설명력)도 경계가 모호.
- 이 겹침을 인정하고 "교차 영역"으로 명시하거나, 축 정의를 더 엄밀하게 분리해야 함.

### 개선 제안
- Y축에 C11(일관성/재현성)을 추가하고, pass^k 방식으로 자동 측정
- X축과 Y축의 겹침을 매핑 테이블로 명시하여 "이 조합은 측정 불필요" 표시
- 시간 효율성은 별도 축보다 모든 문제의 메타데이터(응답 시간)로 기록

## 2. 실행 가능성

### 강점
자동 채점 가능 비율 40%는 현실적이며, JSON 형식의 문제 구조가 파이프라인 자동화에 적합하다.

### 약점/문제점

**인력 병목:**
- 수동 채점 항목(C2, C5, C6, C10)은 전문가 평가 필요 → 현재 팀 3명으로는 도메인당 10~20문제 × 5모델 = 50~100건의 수동 평가 불가능
- 평가자 간 일치도(Inter-rater reliability) 확보 방법 미제시

**비용 병목:**
- 5모델 × 80문제 = 400회 API 호출. 고가 모델(claude-opus $5.5/27.5) 포함 시 1회 벤치마크 실행 비용이 상당
- 학생 개인 키로 실험 시 비용 부담

**기술 병목:**
- 멀티턴 평가 인프라 미구축 — 현재 Playground는 단일 턴만 지원
- 자동 채점 파이프라인(must_include/exclude 체크) 코드 미작성

### 개선 제안
- 1차: 자동 채점 가능한 C3, C4, C8, C9만으로 "자동 벤치마크 v0.1" 먼저 출시
- 수동 채점은 "학생 자기 평가 + 멘토 검증" 2단계로 비용 절감
- 비용 추정표를 미리 작성하여 예산 계획 수립

## 3. 차별성

### 강점
"학생 관점", "한국어 학술 문체", "가성비(CER)" 세 가지가 기존 벤치마크에 없는 명확한 차별점이다.

### 약점/문제점

- **OpenLearnLM과의 차이가 불명확** — OpenLearnLM도 Knowledge-Skill-Attitude 3축으로 교육 LLM을 평가함. 우리와 뭐가 다른지 한 문장으로 설명할 수 있어야 함.
- **SHAPE와의 차이** — SHAPE도 "학습 유도 vs 정답 제공" 균형을 측정. 우리 C6과 거의 동일.
- **KoNET과의 차이** — KoNET도 한국 교육 표준 기반. 우리는 "대학 과제"에 집중한다는 점이 차이지만, 이걸 더 선명하게 해야 함.

**핵심 질문:** "왜 기존 벤치마크를 조합해서 쓰면 안 되는가?"에 대한 답이 약함.

### 개선 제안
- 차별점을 한 문장으로: "wemeet-uni-bench는 한국 대학생이 실제 과제를 할 때 어떤 AI 모델이 가장 도움이 되는지를, 적정 비용으로 알려주는 유일한 벤치마크다."
- 기존 벤치마크와의 "비교 불가능한 영역"을 명시: 한국어 서술형 + 가성비 + 학습 유도 조합은 어디에도 없음

## 4. 스케일링 타당성

### 강점
Bloom's Taxonomy 매핑이 교육학적으로 표준적이며, 난이도 조절 변수(제약 수, 추론 단계, 용어 수준)가 구체적이다.

### 약점/문제점

- **L0~L4가 선형적이라는 가정** — 실제로 L2→L3 전환은 "양적 증가"가 아니라 "질적 전환"(적용→분석). 같은 변수를 늘리는 것만으로는 레벨 전환이 안 될 수 있음.
- **Bloom 매핑의 한계** — Bloom은 1956년(개정 2001년) 프레임워크. AI 시대에 "기억"은 의미 없고, "AI와 협업하여 창조"라는 새로운 단계가 필요할 수 있음.
- **검증 부재** — L2 문제와 L3 문제를 실제로 모델에 돌렸을 때 점수 차이가 유의미한지 아직 검증 안 됨.

### 개선 제안
- 김태운 학생의 기존 문제 중 2~3개를 L1/L2/L3 버전으로 변형하여 파일럿 검증
- Bloom 대신 SOLO Taxonomy(Structure of Observed Learning Outcomes) 병행 검토 — 더 세밀한 질적 단계 구분 가능
- "AI 시대 Bloom" 재해석: 기억→AI에 위임, 이해→AI 출력 검증, 적용→AI와 협업, 분석→AI 한계 파악, 평가→AI 출력 비판, 창조→AI를 도구로 새로운 것 생성

## 5. 한국 특수성

### 강점
한국어 학술 문체, 코드스위칭, 2022 개정 교육과정 매핑이 포함되어 있다.

### 약점/문제점

- **입시 체제 미반영** — 한국 대학생의 현실: 학점(GPA), 대학원 진학, 취업. "과제 도움"뿐 아니라 "시험 대비", "학점 관리 전략"도 AI에게 묻는 상황.
- **한국 학술 문화 특수성** — 교수-학생 위계, 조교 피드백 문화, "정답이 있는 과제" vs "열린 과제" 비율, 팀 프로젝트 문화. 이런 맥락이 문제 설계에 반영되어야 함.
- **한국어 AI 성능의 구조적 약점** — 대부분의 LLM이 영어 중심 학습. 한국어 전문 용어의 정확도가 영어보다 낮을 수 있음. 이걸 "발견"하는 것 자체가 벤치마크의 가치.
- **수능/내신 연계** — 고등학생 확장(Z축 L1) 시 수능 유형과의 연계 필요. 현재 프레임워크에 이 고려 없음.

### 개선 제안
- "한국 대학 과제 유형 분류" 추가: 정답형 과제 / 서술형 레포트 / 팀 프로젝트 / 발표 / 시험 대비
- 한국어 전문 용어 정확도를 C1(정확성)의 하위 지표로 명시
- 향후 고등학생 확장 시 수능 유형 매핑 로드맵 포함

## 6. 가성비 측정 (CER)

### 강점
"적정 비용 · 적정 성능"이라는 핵심 가치를 정량화하려는 시도 자체가 차별적이다.

### 약점/문제점

- **CER 공식이 너무 단순** — `SHS / (cost × tokens)`는 모든 도메인과 능력을 동일 가중치로 봄. 학생 입장에서 "코딩 정확성 90점"과 "문체 적합성 90점"의 가치가 같지 않음.
- **토큰 수 변동** — 같은 문제라도 모델마다 출력 길이가 다름. 긴 답변이 항상 좋은 건 아님.
- **숨겨진 비용** — API 호출 비용 외에 "학생이 AI 답변을 검증하는 시간"도 비용. 부정확한 답변은 검증 비용이 높음.
- **무료 모델 처리** — ChatGPT 무료 버전, Gemini 무료 등은 CER이 무한대. 이걸 어떻게 처리?

### 개선 제안
- CER을 도메인별로 분리: "코딩 CER", "수학 CER" 등
- "검증 비용" 개념 도입: 정확성이 낮으면 CER에 페널티
- 무료 모델은 별도 카테고리로 분리 ("무료 중 최선" vs "유료 중 최선")
- 학생 설문으로 "이 답변에 얼마까지 지불할 의향이 있는가" WTP(Willingness to Pay) 조사

## 7. 지속 가능성

### 강점
"살아있는 데이터셋"이라는 비전이 명확하고, WE-Meet 프로그램 구조(매 학기 새 코호트)가 자연스러운 갱신 메커니즘을 제공한다.

### 약점/문제점

- **문제 품질 관리** — 학생이 만든 문제의 품질이 균일하지 않을 수 있음. 김태운 학생은 매우 높은 수준이지만, 모든 학생이 그렇지는 않음.
- **문제 유출/오염** — 공개 벤치마크의 숙명. 모델 학습 데이터에 우리 문제가 포함되면 벤치마크 무효화.
- **인프라 유지** — GitHub Pages + SAM API 의존. SAM 서비스 중단 시 벤치마크 실행 불가.
- **동기 부여** — 1기 이후 학생들이 계속 문제를 만들 동기가 있는가? 학점 인정이 끝나면?

### 개선 제안
- 문제 품질 기준(rubric 완성도, 자동 채점 가능 비율 등) 정의 → 기준 미달 문제 필터링
- 문제 일부를 비공개(private set)로 유지하여 오염 방지
- SAM 외 대체 실행 경로(OpenAI API 직접 호출 등) 확보
- "벤치마크 기여 = 논문 공저" 같은 학술적 인센티브 설계

## 8. 윤리적 고려

### 강점
C6(학습 유도)를 별도 축으로 두어 "AI가 답을 줘버리는 문제"를 인식하고 있다.

### 약점/문제점

- **측정 자체의 역설** — "AI가 학업에 도움이 되는가"를 측정하면, 결과적으로 "AI를 더 많이 쓰라"는 메시지가 됨. 학생의 자기주도 학습 능력 약화 우려.
- **공정성 문제** — AI를 잘 활용하는 학생 vs 못하는 학생 간 격차 확대. 벤치마크가 이 격차를 정당화하는 도구가 될 수 있음.
- **학술 윤리** — AI 활용 과제의 표절/부정행위 경계가 모호. 벤치마크가 "이 모델로 과제하면 좋다"를 알려주면 부정행위 조장?
- **데이터 프라이버시** — 학생이 만든 문제에 개인 정보(수업명, 교수명 등)가 포함될 수 있음.

### 개선 제안
- 벤치마크 결과 발표 시 "AI 활용 가이드라인" 동시 제공 (어디까지가 적절한 활용인가)
- C6(학습 유도) 점수가 높은 모델을 "학습용 추천", 낮은 모델을 "작업용 추천"으로 구분
- 문제 제출 시 개인정보 제거 가이드라인 제공
- "AI 의존도 자가 진단" 도구를 벤치마크와 함께 배포

## 우선 개선 사항 (Top 5)

1. **자동 채점 파이프라인 먼저 구축** — C3, C4, C8, C9만으로 "v0.1 자동 벤치마크" 출시
2. **Y축에 C11(일관성) 추가** — pass^k 방식, 자동 측정 가능
3. **CER 공식 도메인별 분리** + 무료 모델 별도 카테고리
4. **파일럿 스케일링 검증** — 기존 문제 2~3개를 L1/L2/L3로 변형하여 실제 점수 차이 확인
5. **문제 품질 기준 정의** — 최소 rubric 항목 수, 자동 채점 가능 비율 등

## 추후 멀티모델 검증 계획

SAM API 키 갱신 후 다음 모델들에게 동일한 비판적 리뷰 요청 예정:
- claude-sonnet-4.6 (추론 강점)
- gpt-5.4 (범용 강점)
- deepseek-v3.2 (가성비 관점)
- claude-opus (최고 품질 관점)

각 모델의 리뷰를 비교하여 "모델마다 다르게 보는 약점"을 추출하면, 프레임워크의 편향을 발견할 수 있음.
