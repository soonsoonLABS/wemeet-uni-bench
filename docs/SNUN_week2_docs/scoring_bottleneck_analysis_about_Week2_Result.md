> **분석 대상**: `cs_kim_taewoon_001` ~ `011` (11개) + `mat_kim_hoyoon_001` ~ `002` (2개) = **총 13개 문제**
> **분석 기준**: 2주차 보고서 §04 비판적 평가 — 능력 축(C1~C10)의 자동/수동 채점 가능 여부

## 프레임워크 기준 정리

2주차 보고서에서 정의한 능력 축 10개 중, **자동 채점이 가능한 축**과 **전문가 수동 채점이 필요한 축**은 다음과 같다.

| 구분 | 능력 축 | 채점 방식 | 자동화 근거 |
|:---:|:---|:---:|:---|
| ✅ | C3 · 제약 준수 (CMP) | **자동** | `must_include` / `must_not_include` 문자열 매칭 |
| ✅ | C4 · 함정 감지 (ROB) | **자동** | `must_not_include`로 동조/환각 패턴 차단 검증 |
| ✅ | C8 · 형식 이행 (FMT) | **자동** | 출력 형식(코드만, 마크다운 블록만 등) 패턴 매칭 |
| ✅ | C9 · 자기 한계 인정 (HON) | **자동** | `must_include`("존재하지", "확인되지" 등) 매칭 |
| ❌ | C1 · 사실 정확성 (ACC) | **수동** | 도메인 전문 지식 필요 |
| ❌ | C2 · 다단계 추론 (RSN) | **수동** | 논리 체인·알고리즘 정확성 검증 필요 |
| ❌ | C5 · 설명력 (EXP) | **수동** | 설명의 논리적 흐름과 깊이 판단 필요 |
| ❌ | C6 · 학습 유도 (PED) | **수동** | 소크라테스식 질문의 적절성 판단 필요 |
| ❌ | C7 · 문체 적합성 (STY) | **수동** | 한국어 품질·톤 앤 매너 판단 필요 |
| ❌ | C10 · 비판적 피드백 (CRT) | **수동** | 피드백의 질적 수준 판단 필요 |

> [!IMPORTANT]
> 보고서에 따르면 자동 채점이 가능한 축은 **40% (C3, C4, C8, C9)** 이며, 나머지 **60%는 전문가 수동 채점**이 필요하다. 특히 C2(추론), C5(설명력), C10(비판적 피드백)은 명시적으로 병목으로 지목되었다.

**📋 문제 (프롬프트)**
> Java를 사용하여 제네릭(Generic) 기반의 최소 힙(Min-Heap) 자료구조인 `CustomMinHeap<T extends Comparable<T>>` 클래스를 구현하시오.
> - 패키지 및 임포트 금지 (`java.util.*`, `java.lang.reflect.*` 등 전면 금지)

| 출력형식_이행 | C8 (FMT) | ✅ **자동** | 코드 스니펫만 출력했는지 패턴 매칭 가능 |

> **병목 수준: 🟡 중간** — Rubric 4항목 중 1항목(알고리즘_효율성)이 완전 수동, 1항목(정확도)이 반자동. 자동화율 약 **50%**.

### 병목 패턴 분석

#### 패턴 A: 자동화 친화적 (🟢 3개 — 010, 011, 008)
- **공통 특징**: "AI가 하지 말아야 할 것"을 측정하는 함정/환각 감지 유형
- **자동화 근거**: `must_include`/`must_not_include`로 동조·환각·레거시 패턴을 정밀하게 차단하는 설계가 이미 완료됨
- **3주차 적용**: 자동 채점 v0.1 파이프라인의 **1순위 적용 대상**

#### 패턴 B: 부분 자동화 가능 (🟡 3개 — 001, 006, 007)
- **공통 특징**: 코드 생성 또는 수학적 정확성 측정 유형
- **병목 원인**: 키워드 존재 여부(자동)는 확인 가능하나, 알고리즘 로직/수식 유도 과정의 **논리적 정합성**(C2)은 실행 테스트 또는 전문가 검토 필요
- **개선 제안**: 코드 문제(001)는 테스트 케이스 기반 자동 실행 검증 도입 가능

#### 패턴 C: 수동 채점 불가피 (🔴 7개 — 002, 003, 004, 005, 009, H-001, H-002)
- **공통 특징**: 설명력(C5), 학습 유도(C6), 비판적 피드백(C10) 측정이 핵심인 유형
- **병목 원인**: "설명이 얼마나 잘 되었는가", "질문이 교육적으로 적절한가", "톤이 학술적인가"와 같은 **질적 판단**이 필수적
- **가장 심각**: 002 (Verilog)와 H-001(전위)은 자동화율 ~12~17%로 거의 전 항목이 수동
- **김호윤 문제 특수성**: H-002(ochem)는 수동 병목이 높지만, `must_not_include` 설계가 정밀하여 C3 축 단독으로는 자동 채점 데모 가치가 높음

### 핵심 시사점

> [!WARNING]
> 13개 문제 중 **7개(53.8%)가 수동 채점 병목이 높은 🔴 등급**이다. 이들은 주로 설명력(C5), 학습 유도(C6), 비판적 피드백(C10)을 측정하는 문제들로, 보고서 §04에서 지적한 "C2·C5·C10 30% 수동 병목"과 정확히 일치한다.

> [!TIP]
> **3주차 자동 채점 v0.1 적용 우선순위 제안:**
> 1. **즉시 적용 가능**: 010 (SOTP), 011 (Spring), 008 (힙 함정) — `must_include`/`must_not_include` 파이프라인만으로 핵심 rubric 커버 가능
> 2. **C3 제약 준수 데모**: H-002 (유기화학) — `must_not_include` 5개가 정밀하여 C3 축 단독 자동 채점의 최적 시연 대상
> 3. **테스트 케이스 보강 시 가능**: 001 (MinHeap), 003 (Cache) — 코드 실행 기반 자동 검증 추가 필요
> 4. **LLM-as-Judge 검토 대상**: 004 (Builder), 005 (프롬프트), 009 (멘토), H-001 (전위) — C5/C6/C10 채점에 LLM 평가자 도입 가능성 검토

> [!NOTE]
> **김호윤 문제에 대한 개선 제안:**
> - **H-001 (전위)**: `must_not_include`가 비어 있으므로, 자동 채점 가능 범위를 넓히려면 금지 패턴 추가가 필요하다. 예: "전위는 뼈대입니다", "비유하자면" 등 비학술적 표현 차단.
> - **H-002 (유기화학)**: 이미 `must_not_include`가 우수하게 설계되어 있으므로, `must_include` 항목을 rubric 단계별로 세분화하면 자동화율을 ~30%에서 ~45%까지 끌어올릴 수 있다.
