# 프로젝트 비전 및 취지

---

## 왜 이 프로젝트를 하는가

### 문제 인식

AI 모델의 성능을 측정하는 벤치마크는 수십 개가 존재하지만, 정작 **한국 대학생이 학업에서 AI를 쓸 때 "어떤 모델이 나한테 도움이 되는가"를 알려주는 지표는 없다.**

- MMLU 90점이면 내 선형대수 과제를 잘 도와주나? → 모름
- HumanEval 95%면 내 자료구조 과제를 잘 풀어주나? → 모름
- GPQA 92%면 내 재료역학 문제를 설명해주나? → 모름

기존 벤치마크는 AI 연구자들이 모델 간 우열을 가리기 위해 만든 것이지, **학생이 실제로 도움받을 수 있는지를 알려주기 위해 만든 것이 아니다.**

더 중요한 문제: 학생은 무한한 예산이 없다. **자신의 목적에 맞는 적정 수준의 모델을 적절한 비용으로 선택**할 수 있어야 한다. 가장 비싼 모델이 항상 최선은 아니다.

---

## 두 가지 핵심 질문

이 프로젝트는 두 가지 질문에서 출발한다:

> **1. 이 AI는 대학 수준의 과제·프로젝트·논문에 실제로 도움이 되는가?**
>
> **2. 이 AI는 학생이 스스로 배우고 성장하도록 잘 지도하는가?**

우리는 이 두 질문을 기준으로 AI의 새로운 **학생 도움도 점수(Student Helpfulness Score)**를 만든다.

## 2트랙 벤치마크 구조

```
wemeet-uni-bench
├─ Track 1: Academic Task Benchmark (대학 실전 과제 벤치마크)
│   "이 AI는 내 과제·프로젝트·논문에 실제로 도움이 되는가?"
│
└─ Track 2: Learning Assistant Benchmark (학습 어시스턴트 벤치마크)
    "이 AI는 내가 더 잘 배우도록 도와주는가?"
```

### Track 1: 대학 실전 과제 벤치마크 (Academic Task Benchmark)

> 핵심: **결과물** 중심 평가

**대상**: 대학생, 학부 2~4학년, 전공 과제를 수행하는 학생

**평가 항목**:
- 전공 개념 이해 및 문제 풀이 정확도
- 풀이 과정 설명력
- 코딩 과제 해결력
- 리서치 보조 능력
- 논문 요약·비판·구조화 능력
- 레포트 작성 보조 능력
- 한국어 학술 문체 품질

**핵심 질문**: "내 과제에 쓸 수 있나?"

**AI의 역할**: 전공 과제 조력자

### Track 2: 학습 어시스턴트 벤치마크 (Learning Assistant Benchmark)

> 핵심: **학습 과정** 중심 평가

**대상**: 중·고등학생, 대학 신입생, 비전공 입문자, 자율 학습자

**평가 항목**:
- 개념을 쉽게 설명하는 능력
- 학생 수준에 맞춰 설명을 조절하는 능력
- 틀린 이해를 바로잡는 능력
- 스스로 생각하게 유도하는 능력 (과도한 정답 제공 대신 힌트)
- 학습 계획을 세워주는 능력
- 동기부여와 피드백 품질
- 안전하고 책임 있는 학습 지도

**핵심 질문**: "내가 더 잘 배우게 도와주나?"

**AI의 역할**: 학습 코치·튜터

### 두 트랙의 핵심 차이

| 구분 | Track 1: 대학 실전 | Track 2: 학습 어시스턴트 |
|------|-------------------|----------------------|
| 핵심 대상 | 대학생, 전공 학습자 | 10대~20대 초반 학습자 |
| 핵심 목적 | 과제·프로젝트·논문에 도움 | 자율 학습·개념 이해·학습 지도 |
| 평가 관점 | 결과물의 정확도와 활용성 | 설명력, 지도력, 학습 효과 |
| 대표 과제 | 전공 문제, 코딩, 리서치, 레포트 | 개념 설명, 힌트, 학습 계획, 피드백 |
| AI의 역할 | 전공 과제 조력자 | 학습 코치·튜터 |
| 핵심 질문 | "내 과제에 쓸 수 있나?" | "내가 더 잘 배우게 도와주나?" |

### 점수 체계

```
Student Helpfulness Score (학생 도움도 종합 점수)
├─ Academic Task Score (대학 실전 도움 점수)
└─ Learning Assistant Score (학습 지도 도움 점수)
```

초기에는 두 점수를 따로 보여주는 것이 더 신뢰감 있다.

## SAM 플랫폼 연동: 기존 점수 vs 실제 학생 체감

### SAM이란

순순팩토리에서 제공하는 AI 라우팅 서비스. 현존하는 대부분의 AI API를 하나의 인터페이스로 활용할 수 있다.

SAM에는 이미 공개 벤치마크 점수를 기반으로 정리한 **모델 랭킹 점수표**가 존재한다:

| # | 모델 | OVR | CHAT | CODE | RSN | SPD | $/M (in/out) |
|---|------|-----|------|------|-----|-----|-------------|
| 1 | Claude Opus 4.7 | 94 | 95 | 96 | 93 | 25 | $5.5/27.5 |
| 2 | GPT-5.4 Pro | 92.7 | 94 | 94 | 96 | 12 | $15/120 |
| 3 | Claude Opus 4.6 | 91.35 | 94 | 94 | 89 | 43 | $5.5/27.5 |
| 4 | Gemini 3.1 Pro Preview | 90.75 | 94 | 90 | 93 | 50 | $2/12 |
| 5 | GPT-5.5 | 89.75 | 92 | 91 | 89 | 50 | $5/30 |
| 6 | Claude Sonnet 4.6 | 87.8 | 92 | 90 | 84 | 53 | $3.3/16.5 |
| 7 | Kimi K2.6 | 84 | 87 | 93 | 75 | 43 | $0.6/2.5 |
| 8 | GPT-5.4 | 83.6 | 89 | 84 | 79 | 64 | $2.5/15 |
| 9 | GLM-5 | 75.15 | 77 | 79 | 68 | 29 | $1/3.2 |
| 10 | DeepSeek V4 Flash | 72.7 | 77 | 78 | 65 | 88 | $0.14/0.28 |
| 11 | SCODE 1.0 | 71.4 | 72 | 82 | 56 | 40 | $3.7/18.5 |
| 12 | DeepSeek V3.2 | 68.55 | 72 | 73 | 61 | 68 | $0.62/1.85 |

### 우리가 검증하려는 것

**"이 점수가 실제 학생 체감과 일치하는가?"**

- OVR 94점인 Claude Opus 4.7이 정말 학부 과제에서도 1등인가?
- $0.14/0.28인 DeepSeek V4 Flash가 간단한 과제에는 충분하지 않은가?
- CODE 93점인 Kimi K2.6이 실제 학부 코딩 과제에서도 그만큼 좋은가?
- 비싼 모델($15/120)과 저렴한 모델($0.62/1.85)의 실제 학생 체감 차이는?

### 핵심 가치: 적정 비용으로 적정 성능

> **학생에게 가장 중요한 것: 자신의 목적에 맞는 적정한 수준의 리소스를 선택하여 '적절'한 비용을 지불하면서 공부 및 연구를 할 수 있는가**

- 모든 과제에 $5.5/27.5 모델을 쓸 필요는 없다
- 간단한 개념 질문에는 $0.14/0.28 모델로 충분할 수 있다
- 논문 수준 리서치에는 비싼 모델이 필요할 수 있다
- **"이 과제에는 어떤 모델이 가성비 최선인가?"** — 이걸 알려주는 것이 목표

결국 우리 벤치마크는 단순히 "어떤 모델이 제일 좋은가"가 아니라, **"내 상황에서 어떤 모델이 가장 합리적인 선택인가"**를 알려주는 가이드가 된다.

## 우리가 만들고 싶은 것

### 학생이 직접 참여하는 살아있는 벤치마크

단순히 연구팀이 문제를 만들고 끝나는 것이 아니라, **실제 학생들이 자신의 과제, 시험 문제, 학습 중 막힌 질문을 계속 제공하면서 성장하는 벤치마크**를 지향한다.

```
학생이 문제 제공 → 벤치마크에 반영 → AI 모델 평가 → 결과 공개 → 학생이 더 나은 모델 선택
     ↑                                                              |
     └──────────────────── 피드백 루프 ─────────────────────────────┘
```

## 핵심 취지

### 1. 🎯 학생 중심의 실질적 평가

> "이 AI가 내 과제에 도움이 되는가?"

기존 벤치마크가 묻는 질문: "이 모델이 얼마나 똑똑한가?"
우리가 묻는 질문: **"이 모델이 나한테 얼마나 도움이 되는가?"**

- 정답을 맞히는 것뿐 아니라, 설명을 잘 하는가
- 풀이 과정이 이해하기 쉬운가
- 한국어로 자연스럽게 답하는가
- 모르는 것을 모른다고 말하는가

### 2. 💰 적정 비용, 적정 성능

> "가장 비싼 모델이 항상 최선은 아니다"

- 과제 유형별 최적 모델 추천
- 비용 대비 성능(가성비) 분석
- 학생 예산에 맞는 모델 선택 가이드
- SAM 기존 점수와 실제 학생 체감 비교 검증

### 3. 🤝 학생이 만들고 학생이 쓰는 공공재

> "내가 어려웠던 문제가 다른 학생에게도 도움이 된다"

- 학생들이 자신이 풀다가 막힌 문제, 실제 시험 문제, 과제에서 AI에게 물어본 질문을 제공
- 이 문제들이 벤치마크에 반영되어 AI 모델의 "학생 도움 능력"을 측정
- 결과가 공개되어 모든 학생이 자기 상황에 맞는 모델을 선택할 수 있음
- **기여한 학생은 크레딧을 받고, 벤치마크는 계속 풍부해짐**

### 4. 📈 계속 성장하는 살아있는 데이터셋

> "한 번 만들고 끝나는 벤치마크가 아니라, 매 학기 업데이트되는 벤치마크"

기존 벤치마크의 고질적 문제:
- 한 번 공개되면 학습 데이터에 오염됨
- 시간이 지나면 포화되어 변별력 상실

우리의 해결책:
- 매 학기 새로운 문제가 학생들로부터 유입
- 자연스럽게 오염 방지 (새 문제가 계속 추가되니까)
- 시대/교육과정 변화에 맞춰 진화

### 5. 🇰🇷 한국 교육 맥락에 맞는 평가

> "한국 대학에서 실제로 쓰이는 방식으로 평가한다"

- 한국어 학술 문체로 답변하는 능력
- 한국 대학 교재/커리큘럼 기반 문제
- 한국 학생이 실제로 AI에게 묻는 방식의 프롬프트
- 한국 교수가 기대하는 답안 형식에 맞는 출력

### 6. 🔬 SAM 플랫폼을 통한 실증 검증

> "공개 벤치마크 점수와 실제 학생 체감을 비교한다"

- SAM의 AI 라우팅을 통해 현존하는 대부분의 AI API를 직접 검증
- 기존 공개 벤치마크 점수(OVR, CHAT, CODE, RSN)와 학생 벤치마크 결과 비교
- 점수 괴리가 큰 모델/카테고리 발견 → 학생에게 실질적 가이드 제공

### 7. 🌐 오픈소스로 누구나 참여 가능

> "서울대에서 시작하지만, 모든 대학의 학생이 기여할 수 있다"

- 문제 제출 → 검수 → 반영 파이프라인을 오픈
- 다른 대학, 다른 전공의 학생도 자기 분야 문제를 추가
- 대학별/전공별 세분화된 결과 제공 가능
- 궁극적으로 "한국 대학생 AI 활용 생태계"의 공공 인프라

## 참여 방법 (학생 기여자)

### 문제 제공하기
1. 자신이 풀다가 막힌 문제, AI에게 물어봤던 질문을 정리
2. GitHub Issue로 제출 (템플릿 제공)
3. 검수 후 벤치마크에 반영
4. 기여자로 크레딧 등록

### 기여 가능한 것들
- 실제 과제/시험에서 가져온 문제 (저작권 주의)
- AI에게 물어봤는데 답이 별로였던 질문
- "이런 건 잘 해주면 좋겠다" 시나리오
- 채점 기준에 대한 피드백
- 기존 문제의 난이도/품질 평가
