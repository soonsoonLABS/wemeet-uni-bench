# LLM 벤치마크 통합 조사 보고서

> 주요 LLM 서비스(OpenRouter, Artificial Analysis, LMArena 등)에서 사용하는 벤치마크를 조사하고,
> 문제 풀이 기반 벤치마크의 특징과 대학생 학업과의 괴리를 분석한다.

## 1. 주요 서비스들이 사용하는 벤치마크 수치

### 서비스별 채택 벤치마크

|--------------|-------------------|

## 2. 각 벤치마크 상세 분석

### 2.1 MMLU / MMLU-Pro (지식 평가)

**개요**
- MMLU: 57개 학문 분야, 16,000+ 4지선다 객관식
- MMLU-Pro: 14개 분야, 12,000문항, 10지선다로 강화

**평가 방법**
- Few-shot (5-shot) 또는 Zero-shot으로 객관식 정답 선택
- 정답률(%) 단일 수치로 보고

**문제 풀이 활용 여부**: ✅ 문제 풀이 기반
- 실제 시험 문제 형식 (객관식)
- 수학, 물리, 화학, 법학, 의학 등 다양한 분야

**한계점**
- 2026년 기준 프론티어 모델 88%+ → 변별력 상실
- Wikipedia 등에서 57% 문항에 오류 발견 (Nature 연구)
- 4지선다 → 찍기 확률 25%, 실제 이해도 측정 한계
- 영어 중심, 한국 대학 맥락 부재

**대학생 학업과의 괴리**
- 객관식만 존재 → 실제 대학 과제는 서술형, 증명, 코딩 프로젝트
- 단답형 지식 확인 → 과정(풀이 과정) 평가 불가
- 난이도가 "전문가 자격시험" 수준으로 편향 → 학부 2~3학년 수준 부재

---

### 2.2 GPQA Diamond (대학원급 추론)

**개요**
- 198문항, 생물/물리/화학 분야
- PhD 전문가가 작성, "Google-proof" 설계
- 비전문가(인터넷 검색 허용)도 34%만 정답

**평가 방법**
- Zero-shot 또는 Few-shot Chain-of-Thought
- 4지선다 객관식

**문제 풀이 활용 여부**: ✅ 문제 풀이 기반
- 깊은 추론이 필요한 과학 문제
- 단순 검색으로 풀 수 없도록 설계

**한계점**
- 198문항으로 매우 적음 → 통계적 분산 큼
- 대학원 이상 난이도 → 학부생에게는 너무 어려움
- 생물/물리/화학 3개 분야만 → 공학, CS, 수학 등 미포함
- 영어 전용

**대학생 학업과의 괴리**
- 난이도: PhD 수준 ↔ 학부 2~4학년 수준 필요
- 분야: 순수과학 3개만 ↔ 공학, CS, 인문사회 등 다양한 전공 필요
- 형식: 객관식만 ↔ 실제는 풀이 과정 서술 필요

---

### 2.3 HLE — Humanity's Last Exam (최고 난이도)

**개요**
- 2,500문항, 100+ 학문 분야
- 도메인 전문가가 "인류 지식의 경계"에서 출제
- 2026년 Nature 게재

**평가 방법**
- Closed-book (검색 불가)
- 객관식 + 단답형 혼합

**문제 풀이 활용 여부**: ✅ 문제 풀이 기반
- 다단계 추론 필요
- 전문가 수준 문제 해결

**현재 성적**
- 최고 모델: ~37-65% (인간 전문가: ~90%)
- 아직 포화되지 않은 유일한 고난도 벤치마크

**대학생 학업과의 괴리**
- 극단적 고난도 → 학부 수준과 완전히 동떨어짐
- "인류 최후의 시험" 컨셉 → 일상적 학업 도움과 무관
- 학생이 AI에게 기대하는 것: 과제 도움, 개념 설명 ↔ HLE가 측정하는 것: 최첨단 지식

---

### 2.4 HumanEval / MBPP (코딩)

**개요**
- HumanEval: 164개 Python 함수 완성 문제
- MBPP: 974개 기초 Python 프로그래밍 문제

**평가 방법**
- pass@k: k개 생성 중 1개라도 테스트 통과하면 성공
- 숨겨진 unit test로 자동 채점

**문제 풀이 활용 여부**: ✅ 코드 문제 풀이 기반
- 함수 시그니처 + docstring → 구현 완성
- 자동 실행 및 테스트 검증

**한계점**
- 164문항으로 적음, 데이터 오염(training data에 포함) 심각
- 단일 함수 수준 → 실제 프로젝트/과제와 괴리
- Python만 → Java, C, C++ 등 대학 수업 언어 미포함
- 2026년 기준 프론티어 모델 95%+ → 변별력 상실

**대학생 학업과의 괴리**
- 알고리즘 과제: 보통 전체 프로그램 작성 ↔ HumanEval은 단일 함수
- 과제 요구사항: 설계, 테스트, 문서화 포함 ↔ 벤치마크는 구현만
- 디버깅, 코드 리뷰, 리팩토링 능력 미측정
- 한국어 주석/설명 능력 미측정

---

### 2.5 SWE-bench (소프트웨어 엔지니어링)

**개요**
- 실제 GitHub 이슈 + PR에서 추출
- 모델이 기존 코드베이스에서 패치를 생성해야 함
- Verified 버전: 500개 검증된 문제

**평가 방법**
- 이슈 설명 + 레포 스냅샷 제공
- 생성된 패치가 FAIL_TO_PASS 테스트 통과 여부로 판정
- 테스트는 모델에게 비공개

**문제 풀이 활용 여부**: ✅ 실제 문제 해결 기반
- 가장 "실무"에 가까운 코딩 벤치마크
- 코드 이해 → 버그 파악 → 수정 패치 생성

**한계점**
- Python 프로젝트 위주
- 스캐폴딩(에이전트 프레임워크)에 따라 점수 25%p 차이
- 대규모 오픈소스 프로젝트 기반 → 학부 과제와 규모 다름

**대학생 학업과의 괴리**
- 실무 수준 ↔ 학부 과제는 소규모 프로젝트
- 기존 대규모 코드베이스 이해 필요 ↔ 학생은 처음부터 작성
- 그러나 "문제 해결" 접근법은 참고할 만함

---

### 2.6 MATH / GSM8K (수학)

**개요**
- GSM8K: 8,792개 초등~중등 수학 문장제
- MATH: 12,500개 경시대회 수준 수학 문제 (7개 분야, 5단계 난이도)

**평가 방법**
- 최종 답 정확도 (숫자/수식 매칭)
- Chain-of-Thought 풀이 후 최종 답 추출

**문제 풀이 활용 여부**: ✅ 수학 문제 풀이 기반
- 단계별 풀이 과정 생성
- 최종 답의 정확성으로 채점

**한계점**
- GSM8K: 완전 포화 (프론티어 모델 95%+)
- MATH: 경시대회 스타일 → 대학 수학(미적분, 선형대수) 과제와 다름
- 풀이 "과정"의 질은 평가하지 않음 (답만 채점)
- 증명 문제 미포함

**대학생 학업과의 괴리**
- GSM8K: 너무 쉬움 (초등 수준)
- MATH: 경시대회 트릭 위주 ↔ 대학 수학은 개념 이해 + 응용
- 미적분, 선형대수, 미분방정식, 확률통계 등 대학 핵심 과목 직접 다루지 않음
- 증명(proof) 작성 능력 미측정
- LaTeX 수식 표현의 정확성/가독성 미평가

---

### 2.7 Arena ELO / LMArena (사용자 선호도)

**개요**
- 사용자가 익명 모델 2개의 응답을 비교 투표
- 6M+ 투표, 339개 모델 참여 (2026.4 기준)
- ELO 레이팅 시스템으로 순위 산출

**평가 방법**
- 블라인드 A/B 비교
- 크라우드소싱 투표 집계
- 카테고리별 세분화 (코딩, 수학, 창작 등)

**문제 풀이 활용 여부**: ❌ 직접적 문제 풀이 아님
- 사용자 "선호도" 측정
- 정답 여부가 아닌 응답 품질 비교

**한계점**
- 주관적 선호 → 정확성과 다를 수 있음
- 영어 사용자 편향
- 특정 도메인 성능 예측 불가

**대학생 학업과의 괴리**
- 일반 사용자 선호 ↔ 학생은 "정확한 풀이"가 필요
- 종합 점수 → 특정 과목/과제 유형별 성능 알 수 없음

---

### 2.8 LiveBench (오염 방지 벤치마크)

**개요**
- 정기적으로 새 문제 출제하여 데이터 오염 방지
- 7개 카테고리, 21~23개 태스크
- 객관적 정답 기반 자동 채점 (LLM 판정 불필요)

**평가 방법**
- 최신 데이터 기반 문제 (뉴스, 논문 등에서 추출)
- 검증 가능한 정답으로 자동 채점

**문제 풀이 활용 여부**: ✅ 부분적 문제 풀이
- 수학, 코딩, 추론 등 포함
- 데이터 오염 문제를 구조적으로 해결

**대학생 학업과의 괴리**
- 오염 방지 접근법은 우리 벤치마크에 참고할 만함
- 그러나 여전히 영어 중심, 학부 과제 시나리오 아님

## 3. 대학 수준 특화 벤치마크 (최근 등장)

기존 벤치마크의 한계를 인식하고, 대학 수준에 맞춘 벤치마크가 최근 등장하고 있다:

### 3.1 U-MATH (2024)
- **규모**: 1,100문항, 대학 수학 6개 분야
- **특징**: 대학 교재에서 추출한 미공개 문제, 20% 멀티모달(그래프/도표)
- **형식**: 서술형 (open-ended)
- **결과**: 텍스트 문제 최고 93.1%, 시각 문제 최고 58.5%
- **의의**: 대학 수학 교재 기반이라 학부 수업과 직접 연관

### 3.2 UGMathBench (ICLR 2025)
- **규모**: 5,062문항, 16개 과목, 111개 토픽
- **특징**: 학부 수학 전용, 10가지 답 유형, 문제당 3개 랜덤 버전 (오염 방지)
- **지표**: Effective Accuracy, Reasoning Gap
- **의의**: 가장 체계적인 학부 수학 벤치마크

### 3.3 CUMath (2025)
- **규모**: 2,100문항 (미적분, 선형대수, 미분방정식 등)
- **특징**: 단계별 풀이 포함 → 중간 추론 과정도 평가 가능
- **의의**: 풀이 과정 평가라는 점에서 우리 목표와 가장 유사

### 3.4 MaterialBENCH (2024)
- **규모**: 대학 재료공학 교재 기반 문제
- **형식**: 서술형 + 객관식 혼합
- **의의**: 특정 전공(재료공학) 대학 수준 평가 → 김호윤 전공과 직접 관련

### 3.5 MatSciBench (2025)
- **규모**: 6개 주요 분야, 31개 세부 분야
- **특징**: 3단계 난이도 분류 (풀이 길이 기반)
- **의의**: 재료과학 추론 능력 체계적 평가

## 4. 종합 분석: 기존 벤치마크 vs 대학생 실제 니즈

### 4.1 기존 벤치마크가 측정하는 것

| 능력 | 대표 벤치마크 | 측정 방식 |
|------|-------------|-----------|
| 지식 암기 | MMLU | 객관식 정답률 |
| 전문가급 추론 | GPQA, HLE | 객관식/단답 |
| 코드 생성 | HumanEval | 테스트 통과율 |
| 실무 코딩 | SWE-bench | 패치 성공률 |
| 수학 계산 | MATH, GSM8K | 최종 답 정확도 |
| 종합 선호도 | Arena ELO | 사용자 투표 |

### 4.2 대학생이 실제로 필요한 것 (기존 벤치마크에 없는 것)

| 니즈 | 설명 | 기존 벤치마크 커버 여부 |
|------|------|----------------------|
| 📝 서술형 풀이 과정 | 단계별 풀이를 보여주며 설명하는 능력 | ❌ (답만 채점) |
| 🇰🇷 한국어 학술 품질 | 자연스러운 한국어 학술 문체 | ❌ (영어 중심) |
| 📚 개념 설명력 | 모르는 개념을 이해시키는 능력 | ❌ (미측정) |
| 🔧 디버깅/코드 리뷰 | 기존 코드의 문제점 찾기 | △ (SWE-bench 일부) |
| 📊 데이터 분석 제안 | 실험 데이터 해석 및 시각화 방법 제안 | ❌ (미측정) |
| 📄 레포트 구조화 | 보고서/논문 구조 잡기 | ❌ (미측정) |
| 🎓 난이도 적합성 | 학부 2~4학년 수준 | ❌ (너무 쉽거나 너무 어려움) |
| 🔄 맥락 연속성 | 이전 대화 기반 후속 질문 처리 | ❌ (단일 문제 위주) |
| ⚠️ 오류 인지 | 자신의 답이 틀렸을 때 인정/수정 | ❌ (미측정) |
| 📖 참고문헌 제시 | 관련 자료/논문 안내 | ❌ (미측정) |

### 4.3 난이도 갭 분석

```
[너무 쉬움]                    [적정 범위]                    [너무 어려움]
GSM8K ──────── MMLU ──── ??? ──── MATH ──── GPQA ──── HLE
(초등)        (자격시험)   (학부)   (경시대회)  (PhD)    (전문가 한계)
                            ↑
                     여기가 비어있음!
```

학부 2~4학년 수준의 "적정 난이도" 벤치마크가 구조적으로 부재하다.
최근 U-MATH, UGMathBench, CUMath 등이 이 갭을 채우려 하지만, 수학에 한정되어 있고 한국어/다전공 커버리지는 없다.

## 5. 우리 벤치마크가 차별화될 수 있는 지점

| 차별화 요소 | 설명 |
|------------|------|
| **한국어 네이티브** | 한국 대학 교재/과제 기반, 한국어 품질 평가 포함 |
| **학부 적정 난이도** | L1~L4 (1학년~4학년) 체계적 난이도 |
| **풀이 과정 평가** | 답뿐 아니라 설명의 질, 단계별 논리 평가 |
| **다전공 커버** | CS, 수학, 재료공학, 인문사회 등 |
| **실제 과제 시나리오** | 레포트, 코딩 과제, 실험 분석 등 실제 형식 |
| **유용성 측정** | "맞냐"가 아니라 "도움이 되냐" 관점 |
| **설명력 평가** | 학습 보조 관점에서의 설명 품질 |
