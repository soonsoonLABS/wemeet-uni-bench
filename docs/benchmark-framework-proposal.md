> 작성일: 2025-05-23
> 작성: 송용성 (멘토) + AI 보조

## 1. 프로젝트 배경

### 1.1 문제 의식

기존 LLM 벤치마크(MMLU, GPQA, HumanEval 등)는 "AI가 얼마나 똑똑한가"를 측정하지만, 실제 한국 대학생이 학업에서 AI를 활용할 때 체감하는 "도움이 되는 정도"와 괴리가 크다.

구체적 문제:
- 난이도 스펙트럼에서 "학부 2~4학년" 구간이 구조적으로 비어있음
- 객관식/단답형만 측정 — 실제 과제는 서술형, 코딩 프로젝트, 레포트
- 최종 답만 채점 — 학생은 풀이 과정과 설명을 원함
- 영어 중심 — 한국어 학술 문체는 측정 대상이 아님
- "도움이 되는가"를 묻지 않음 — 맞는 답 ≠ 도움이 되는 답

### 1.2 목표

**대학생 수준(100점 기준)의 종합 학습 도움도 벤치마크**를 만들고, 그 기준을 아래로 스케일링하여 고등→중등→초등까지 확장 가능한 체계를 설계한다.

### 1.3 핵심 질문

> "이 AI는 내 학업에 실제로 도움이 되는가? 그리고 그 도움을 적정한 비용으로 받을 수 있는가?"

## 2. 3축 벤치마크 구조

### 2.1 개념

```
X축 = 도메인 (Domain)     — "무엇을" 도와주는가
Y축 = 능력 (Capability)   — "어떻게" 도와주는가
Z축 = 난이도 (Difficulty)  — "얼마나 깊이" 도와주는가
```

현재 단계에서는 X×Y 평면을 먼저 채우고, Z축(난이도 스케일링)은 학생들이 직접 문제를 만들고 실험하면서 점진적으로 확장한다.

### 2.2 설계 원칙

1. **학생 중심** — 연구자가 아닌 학생의 실제 학업 상황에서 출발
2. **다축 프로파일** — 단일 점수가 아닌 도메인×능력 히트맵으로 결과 제시
3. **가성비 포함** — 같은 점수를 더 싼 모델로 달성 가능한지 측정
4. **한국어 맥락** — 한국어 학술 문체, 한영 코드스위칭, 한국 교육과정 반영
5. **살아있는 데이터셋** — 매 학기 새 문제 유입, 시대에 맞춰 진화
6. **자동+수동 채점 혼합** — 형식 이행은 자동, 설명력/학습유도는 전문가 판단

## 3. X축 — 도메인 (9개)

대학생이 AI에게 실제로 도움을 요청하는 학업 상황을 기준으로 분류.

|---|---|---|---|---|

| D2 | 시스템/HW | SYS | 운영체제, 컴퓨터구조, 회로설계, 임베디드 | Verilog 디버깅, 캐시 시뮬레이터, 동기화 문제 |
| D3 | 수학/공학수학 | MATH | 선형대수, 미적분, 확률통계, 수치해석, 이산수학 | SVD 응용, 증명 문제, 수식 유도 |
| D4 | 자연과학/공학 | SCI | 물리, 화학, 재료공학, 생물, 전자공학 | 전위 분석, 열역학 계산, 실험 데이터 해석 |
| D5 | 인문/사회 | HUM | 경제, 법, 철학, 역사, 심리, 경영 | 논증 구성, 사례 분석, 비판적 에세이 |
| D6 | 논문/리서치 | RES | 문헌 검색, 선행연구 정리, 연구 방법론, 인용 | 논문 요약, 연구 설계, 문헌 리뷰 구조화 |
| D7 | 데이터/통계 | DATA | 실험 설계, 통계 분석, 시각화, 데이터 전처리 | t-test 해석, 회귀분석, 그래프 생성 |
| D8 | 글쓰기/발표 | WRITE | 레포트, 발표자료, 학술 문체 작성, 구조화 | 서론 작성, 슬라이드 구성, 초록 작성 |

### 도메인 분류 원칙

- **전공별 도메인 (D1~D5)**: 특정 전공 지식이 필요한 과제
- **범전공 도메인 (D6~D9)**: 전공 불문 모든 대학생에게 공통으로 필요한 학업 능력
- **확장 가능**: 새로운 전공 도메인은 D10, D11...로 추가 가능

## 4. Y축 — 능력 (10개)

AI가 학생에게 "도움이 되려면" 갖춰야 할 역량을 분해. 김태운 학생의 11개 문제에서 실증된 것 + 글로벌 교육 연구(OECD, Bloom, SHAPE, Pedagogical Taxonomy)에서 도출.

| ID | 능력 | 약칭 | 정의 | 자동 채점 가능 여부 |
|---|---|---|---|---|
| C1 | 사실 정확성 | ACC | 제공하는 정보가 사실적으로 맞는가 | 부분 자동 (정답 대조) |
| C2 | 다단계 추론 | RSN | 복잡한 논리를 단계별로 따라가는가 | 수동 (논리 흐름 검증) |
| C3 | 제약 준수 | CMP | 명시된 조건/금지사항을 정확히 지키는가 | **완전 자동** (must_include/exclude) |
| C4 | 함정 감지 | ROB | 잘못된 전제, 가짜 개념, 폐기된 API를 거부하는가 | **완전 자동** (거부 여부 판정) |
| C5 | 설명력 | EXP | 풀이 과정을 명확하고 단계적으로 보여주는가 | 수동 (전문가 평가) |
| C6 | 학습 유도 | PED | 답을 직접 주지 않고 학생의 사고를 유도하는가 | 반자동 (정답 제공 여부 자동 + 유도 품질 수동) |
| C7 | 문체 적합성 | STY | 한국어 학술 문체, 자연스러운 한영 코드스위칭 | 반자동 (번역체 패턴 탐지 + 수동) |
| C8 | 형식 이행 | FMT | 요구된 출력 형식(LaTeX, 코드만, 구조화)을 따르는가 | **완전 자동** (형식 검증) |
| C9 | 자기 한계 인정 | HON | 모르는 것을 솔직히 인정하는가 (환각 억제) | **완전 자동** (가짜 개념 문제에서) |
| C10 | 비판적 피드백 | CRT | 학생의 오류를 객관적으로 지적하고 교정하는가 | 수동 (교정 정확성 + 톤 평가) |

### 능력 분류 원칙

- **결과물 품질 (C1, C2, C3, C8)**: AI 출력의 객관적 품질
- **교육적 태도 (C5, C6, C10)**: 학습을 돕는 방식의 질
- **신뢰성 (C4, C9)**: 잘못된 정보를 주지 않는 능력
- **문화적 적합성 (C7)**: 한국 교육 맥락에 맞는 표현

### 자동 채점 비율

- 완전 자동: 4/10 (C3, C4, C8, C9)
- 반자동: 2/10 (C6, C7)
- 수동: 4/10 (C1, C2, C5, C10)

→ 40%는 완전 자동화 가능, 나머지는 전문가 또는 학생 평가 필요

## 5. Z축 — 난이도 (향후 확장)

현재는 대학 수준(L2-L3)에 집중하되, 동일한 X×Y 좌표에서 난이도만 조절하여 스케일링.

| 레벨 | 대상 | Bloom 단계 | 난이도 조절 변수 |
|---|---|---|---|
| L0 | 초등 4-6 | 기억/이해 | 제약 0~1개, 단일 단계, 일상어 |
| L1 | 중등 (중1~고1) | 이해/적용 | 제약 1~2개, 2단계 추론, 기초 용어 |
| L2 | 고등~대학초 (고2~대2) | 적용/분석 | 제약 2~3개, 3~4단계, 학술 문체 |
| L3 | 대학 중반~졸업 (대2~4) | 분석/평가 | 제약 3~4개, 5단계+, 전문 용어+코드스위칭 |
| L4 | 대학원/전문가 | 평가/창조 | 복합 제약, 다단계 함정, 최신 연구 맥락 |

### 스케일링 예시 (D1 코딩 × C3 제약 준수)

- **L1**: "Python으로 피보나치 함수를 작성하라"
- **L2**: "Python으로 피보나치를 작성하되, 재귀 금지, O(N) 보장"
- **L3**: "Java로 제네릭 Min-Heap을 구현하되, import 금지, Object[] 사용 강제, 동적 확장 직접 구현" (김태운 001번)
- **L4**: "위 조건 + 스레드 안전성 보장 + 성능 벤치마크 코드 포함"

## 6. 점수 체계

### 6.1 개별 문제 점수

각 문제는 해당하는 능력 축(Y축)에 대해 0~100점으로 채점.

```
문제 점수 = Σ(rubric 항목별 점수) / rubric 항목 수 × 100
```

### 6.2 모델 프로파일 (X×Y 히트맵)

```
모델 A 프로파일:
         D1   D2   D3   D4   D5   D6   D7   D8   D9
C1 정확성 92   85   78   --   --   --   --   --   95
C2 추론   88   80   72   --   --   --   --   --   90
C3 제약   95   90   --   --   --   --   --   --   --
...
```

### 6.3 종합 점수 — Student Helpfulness Score (SHS)

```
SHS = Academic Task Score (60%) + Learning Assistant Score (40%)

Academic Task Score = 가중평균(C1, C2, C3, C8) across all domains
Learning Assistant Score = 가중평균(C5, C6, C9, C10) across all domains
```

### 6.4 가성비 지표 — Cost Efficiency Ratio (CER)

```
CER = SHS / (cost per 1K tokens × average tokens per problem)
```

→ "같은 도움을 받는 데 얼마나 적은 비용이 드는가"

## 10. 예상 한계 및 미해결 과제

1. **채점 객관성**: 수동 채점 항목(C2, C5, C6, C10)의 평가자 간 일치도 확보 방법
2. **문제 수**: 현재 11개 — 통계적 유의성을 위해 도메인당 최소 10~20개 필요
3. **비전공 도메인**: D5(인문/사회), D6(논문), D7(데이터), D8(글쓰기) 문제 부재
4. **멀티턴 평가**: 현재 모든 문제가 단일 턴 — 맥락 유지 능력 미측정
5. **학생 다양성**: 현재 2명(컴공, 재료) — 더 다양한 전공/학년 필요
6. **Z축 검증**: 같은 문제의 난이도 스케일링이 실제로 유의미한 차이를 만드는지 미검증
7. **가성비 측정**: CER 공식의 가중치 설정 근거 부족
8. **시간 경과**: 모델 업데이트 시 벤치마크 결과의 유효기간

## 11. 비판적 검토 요청 사항

이 프레임워크에 대해 다음 관점에서 비판적 평가를 요청합니다:

1. **구조적 완결성**: X축 9개, Y축 10개가 대학생 학업 도움도를 측정하기에 충분한가? 빠진 중요한 축이 있는가?
2. **실행 가능성**: 이 프레임워크로 실제 벤치마크를 운영할 때 현실적 장벽은 무엇인가?
3. **차별성**: 기존 교육 AI 벤치마크(OpenLearnLM, SHAPE, KoNET 등)와 비교했을 때 우리만의 고유한 가치는 충분한가?
4. **스케일링 타당성**: L0~L4 난이도 체계가 교육학적으로 타당한가? Bloom's Taxonomy 매핑이 적절한가?
5. **한국 특수성**: 한국 교육 맥락의 특수성이 충분히 반영되었는가? 빠진 한국 특유의 요소가 있는가?
6. **가성비 측정**: "적정 비용 · 적정 성능"을 측정하는 CER 공식이 학생에게 실질적 가이드를 줄 수 있는가?
7. **지속 가능성**: "살아있는 데이터셋"을 매 학기 갱신하는 것이 현실적으로 가능한가? 어떤 인프라가 필요한가?
8. **윤리적 고려**: 이 벤치마크가 학생의 AI 의존도를 높이는 부작용은 없는가? 학습 유도(C6)만으로 충분한 안전장치인가?
